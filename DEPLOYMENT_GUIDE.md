# 🚀 Deployment Guide - Ubuntu Server 24 LTS

## Prerequisites

- Ubuntu Server 24 LTS
- Docker & Docker Compose installed
- Git installed

## 📋 Quick Start

### 1. Clone Repository

```bash
git clone https://github.com/bintang1855/carbonquest-api.git
cd carbonquest-api
```

### 2. Setup Environment

```bash
# Copy dan edit .env file
cp .env.example .env
nano .env
```

Isi `.env` file:

```env
DATABASE_URL="postgresql://carbonquest:secret@db:5432/carbonquest_db"
JWT_SECRET="your-super-secret-jwt-key-change-this"
PORT=4000
NODE_ENV=production
```

### 3. Build & Run dengan Docker

```bash
# Build image (dengan --no-cache untuk clean build)
docker compose build --no-cache

# Start containers
docker compose up -d

# Check logs
docker compose logs -f app
```

### 4. Database Migration

```bash
# Run migration di dalam container
docker compose exec app npx prisma migrate deploy

# Atau jika ingin create migration baru
docker compose exec app npx prisma migrate dev
```

## 🔄 Update/Rebuild After Code Changes

Setiap kali ada perubahan code di GitHub:

```bash
# Pull latest changes
git pull origin test-api

# Rebuild container
docker compose down
docker compose build --no-cache
docker compose up -d

# Run migrations (if any)
docker compose exec app npx prisma migrate deploy
```

## 📁 Folder Structure

```

├── dist/              # Compiled code (JavaScript) - Generated by build
```

- `db_data` - PostgreSQL data (persistent)
- `uploads_data` - Uploaded images (persistent)

## 🛠️ Useful Commands

```bash
# Check running containers
docker compose ps

# View logs
docker compose logs -f app

# Stop containers
docker compose down

# Restart containers
docker compose restart app

# Execute command in container
docker compose exec app <command>

# Clean up everything (CAUTION: deletes volumes)
docker compose down -v

# Access PostgreSQL
docker compose exec db psql -U carbonquest -d carbonquest_db
```

## 🔍 Troubleshooting

### Container won't start

```bash
# Check logs
docker compose logs app

# Check if port is already in use
sudo netstat -tulpn | grep 4000
```

### Database connection error

```bash
# Verify DATABASE_URL in .env
# Make sure it uses "db" as hostname (not localhost)
DATABASE_URL="postgresql://carbonquest:secret@db:5432/carbonquest_db"

# Restart containers
docker compose restart
```

### Uploads not working

```bash
# Check if uploads volume exists
docker volume ls | grep uploads

# Check permissions inside container
docker compose exec app ls -la /app/uploads
```

### Migration errors

```bash
# Reset database (CAUTION: deletes all data)
docker compose down -v
docker compose up -d
docker compose exec app npx prisma migrate deploy
```

## 🌐 API Access

- **API Base URL**: `http://your-server-ip:4000`
- **Swagger Docs**: `http://your-server-ip:4000/docs`
- **Health Check**: `http://your-server-ip:4000/`
- **Uploaded Images**: `http://your-server-ip:4000/uploads/<filename>`

## 🔐 Security Notes

1. **Change JWT_SECRET** - Use a strong random string
2. **Use HTTPS** - Cloudflare tunnel already configured in docker-compose
3. **Rate Limiting** - Already configured (1000 req/min global, 5 req/15min for auth)
4. **Environment Variables** - Never commit `.env` to Git
